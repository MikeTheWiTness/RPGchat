# ADR-0003: NPC 侧采用运势系统而非骰子检定

**日期**: 2026-06-24  
**状态**: Accepted

## 背景

在最初的设计中，骰子/技能检定系统对所有角色一视同仁——PC 和 NPC 都使用相同的检定机制。但这带来了几个问题：

1. NPC 之间的互动如果每次都走严谨的骰子检定，会拖慢叙事节奏，增加 LLM 调用负担
2. 严格的检定机制可能让 NPC 行为过于机械，缺乏 TRPG 中「NPC 也会犯错误/超常发挥」的生动感
3. 不是所有 LLM 都支持 temperature 参数调节，依赖模型原生参数的方案不够通用

同时，PC 侧确实需要严谨的机制来保证玩家的参与感和公平感。

因此需要重新设计：PC 和 NPC 采用不同的随机性机制。

## 决策

**PC 侧用严谨的 d20 骰子检定，NPC 侧用运势系统 (Fortune System) 软调节。**

### 运势系统核心设计

- **五级制**：大吉 / 吉 / 平 / 凶 / 大凶，由 d100 投点分档
- **按检查点刷新**：每个检查点周期内角色运势固定，避免短时间内大量波动导致叙事混乱
- **角色独立**：每个 NPC 有自己的运势，不同角色状态不同
- **实现方式**：程序生成随机数 → 映射到运势等级 → 在 LLM 提示词中注入行为倾向描述，不依赖 LLM 原生 temperature
- **作用范围**：只影响行为倾向（叙事层面的软调节），不直接决定成败（不是数值硬规则）

### 与骰子检定的关系

| 维度 | 骰子检定（PC 侧） | 运势系统（NPC 侧） |
|------|------------------|-------------------|
| 机制 | d20 + 技能加值 vs DC | d100 → 五级运势 → 提示词引导 |
| 精确性 | 精确，明确成功/失败 | 模糊，影响行为倾向 |
| 刷新频率 | 每次检定一次 | 每个检查点一次 |
| 作用对象 | 玩家（PC） | 所有 NPC |
| 玩家感知 | 玩家主动触发，结果明确可见 | 玩家可观察到 NPC 状态，但不直接操作 |

## 权衡

### 为什么 NPC 侧不用骰子检定

- **叙事流畅性**：NPC 之间的互动是背景叙事，不需要每次都停下来投骰
- **避免数值膨胀**：如果所有 NPC 都需要完整的技能数值，角色档案会很重，生成成本高
- **戏剧多样性**：运势的提示词引导比硬检定更容易产生有趣的意外

### 为什么用 d100 + 五级制而不是其他方案

- **d100 分档灵活**：可以精细调整各等级的概率
- **五级制符合直觉**：玩家对「大吉/大凶」这种分级有文化认知（类似东方命理文化）
- **按检查点刷新**：避免单次波动，保持每个阶段内角色行为的一致性

### 负面后果

- PC 和 NPC 两套机制，理解成本略高
- 运势系统的效果高度依赖提示词工程，不同 LLM 的表现可能有差异
- 玩家可能会疑惑「为什么 NPC 不用投骰子」

## 后果追踪

需要同步更新/开发的内容：
- CONTEXT.md 中「运势系统」词条（已更新）
- 检查点逻辑：增加运势刷新步骤
- NPC 生成提示词：根据运势等级注入行为倾向描述
- UI 展示：在角色面板或状态栏显示各角色当前运势
