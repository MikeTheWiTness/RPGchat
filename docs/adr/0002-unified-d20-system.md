# ADR-0002: 统一采用 d20 骰子系统

**日期**: 2026-06-23  
**状态**: Accepted

## 背景

RPG Chat Agent 的轻量规则集成模块最初设计为可插拔的多系统架构，支持 COC 7th（d100 roll-under）和 D&D 5e（d20 roll-over）等多种规则配置。`CharacterProfile.skills` 中存储的数值含义随规则系统变化，导致：

- 角色档案的技能值语义不统一
- 预设系统（角色预设、世界观预设）需要适配多套规则
- 玩家认知成本高，需要理解不同系统的技能值含义
- 代码中两套逻辑并存，维护成本高

经过评估，决定统一采用一套骰子系统，降低复杂度。

## 决策

**统一使用 d20 系统（roll-over）** 作为轻量规则集成的唯一骰子系统。

核心规则：
- 投 1d20 + 技能加值 >= 难度等级（DC）= 成功
- `CharacterProfile.skills` 中存储的是**技能加值**（modifier）
- 自然 20 = 暴击（critical success）
- 自然 1 = 大失败（fumble）

COC 等其他系统的预设不再作为一等公民支持。未来如有需要，可通过规则转换层适配，但核心数据模型统一为 d20。

## 权衡

### 为什么选 d20 而不是 3d6 / d100

- **d20 vs 3d6**：d20 均匀分布，戏剧性更强，每次投骰都有 5% 的极端概率，更适合 AI 叙事的波动感。3d6 钟形曲线过于稳定，技能差距被放大，不利于叙事多样性。
- **d20 vs d100**：d20 认知成本最低（D&D 是最广为人知的 TRPG 系统）。d100 百分比虽然直观，但需要技能值范围为 1-100，角色成长曲线设计更复杂。

### 正面后果

- 技能值语义统一，角色档案可在不同游戏间复用
- 预设系统只需一套标准，开发和使用成本降低
- 玩家认知成本最低，d20 + DC 的模型最直观
- 代码简化，移除多系统适配分支
- 暴击/大失败各 5% 的概率，叙事起伏适中

### 负面后果

- 失去 COC 等系统的原生支持，喜爱 d100 的玩家需要适应
- 均匀分布意味着技能差距不那么明显（+3 的技能差距在 d20 上是 15% 的胜率差，在 3d6 上差距更大）
- 未来如果要支持其他系统，需要通过转换层适配，可能不够原生

## 后果追踪

需要同步更新的内容：
- CONTEXT.md 中「游戏机制集成」词条（已更新）
- `rules.py` 中的多预设系统（待清理，保留一个 d20 标准配置即可）
- `dice.py` 中的多分支逻辑（待简化）
- 角色预设格式（统一为技能加值）
