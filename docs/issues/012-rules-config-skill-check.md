# Issue #12: 规则配置 + 技能检定集成

## What to build

将骰子引擎集成到游戏流程中，支持纯叙事和轻量规则两种模式。

- **规则配置文件格式**：定义 JSON 规则配置 schema
  - COC 7th 预设：属性（STR/CON/DEX/APP 等）、技能列表、d100 对抗检定
  - D&D 5e 预设：属性（STR/DEX/CON/INT/WIS/CHA）、技能列表、d20 + 调整值 vs DC
- **集成到 PC 输入**：`{检定 侦查}` → 解析为检定指令，调用 DiceEngine，将结果注入动作单元
- **集成到 NPC 生成**：NPC 的 LLM prompt 中包含角色技能值，Agent 可自主决定何时触发检定
- **两种模式开关**：
  - 纯叙事模式：角色档案不需要属性点，忽略检定指令（提示"纯叙事模式不支持检定"）
  - 轻量规则模式：角色档案扩展属性点字段，检定正常执行
- 检定结果格式化展示在剧情流中

## Acceptance criteria

- [ ] COC/D&D 规则配置正确加载
- [ ] `{检定 侦查}` → 骰子执行，结果插入剧情流
- [ ] 纯叙事模式下 `{检定}` 提示不可用
- [ ] NPC 在轻量规则模式下可自动触发检定
- [ ] 检定结果显示格式清晰（如 `d100=45 vs 侦查60 → 成功`）
- [ ] 集成测试覆盖两种模式的检定流程
