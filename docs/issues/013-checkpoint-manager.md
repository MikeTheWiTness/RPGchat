# Issue #13: 检查点管理器 (CheckpointManager)

## What to build

实现检查点触发的上下文压缩和附带操作模块。

- `trigger_checkpoint(session)` — 触发检查点
  1. 调用 LLM 基于当前阶段内容生成摘要（Summary）
     - 摘要结构：该阶段重要事件、角色变化、环境变迁
  2. 将检查点之前的细粒度上下文替换为摘要（原始数据归档或丢弃）
  3. 当前阶段内上下文保持不变
  4. 触发附带操作：
     - **角色档案更新**：调用 LLM 为每个主要角色生成更新后的档案（性格变化、技能成长、关系变更）
     - **环境上下文更新**：清理过期环境条目，写入当前全局环境状态描述
- `restore_from_checkpoint(session)` — 从最新检查点恢复（用于加载存档时快速定位）
- 检查点粒度可配置：按场景、按天、按章节

## Acceptance criteria

- [ ] 触发后当前阶段前的上下文被摘要替代
- [ ] 当前阶段内上下文保持完整
- [ ] 角色档案在检查点后正确更新
- [ ] 过期环境条目被清理
- [ ] 检查点按配置粒度触发
- [ ] 可指定检查点触发条件（手动或自动）
- [ ] 单元测试覆盖上下文压缩前后的数据状态变化
