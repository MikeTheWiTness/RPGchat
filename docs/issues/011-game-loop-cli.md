# Issue #11: 游戏主循环 + CLI 界面

## What to build

实现游戏核心编排器和命令行交互界面。

### 游戏主循环 (GameLoop)
编排完整的游戏流程：
1. 初始化：加载/创建 GameSession
2. 生成初始环境描述（环境第一个动作单元）
3. 判断机制决定第一个角色
4. 进入循环：
   - 等待用户输入（`{继续}` 或 PC 动作单元）
   - 若 `{继续}`：采纳判断机制推荐，生成对应 NPC/环境动作单元
   - 若 PC 输入：解析为 ActionUnit，写入 PC 上下文和 DialogueLog
   - 每个动作单元后：更新 SceneTracker、追加 DialogueLog、运行判断机制
   - 强制环境插入检查（连续角色 >= N 时）
   - 周期性一致性校验（每 K 个动作单元）

### CLI 界面
- 引导流程：创建新游戏或加载存档
- 创建流程：输入战役背景 → 配置模式 → 创建 PC → 开始
- 游戏界面：剧情流展示（角色名 + 对话/行动，内心活动折叠/灰色显示）、输入区、`{继续}` 快捷指令
- 显示在场人员列表（可选切换）

## Acceptance criteria

- [ ] 新游戏创建：输入背景 → 初始环境描述 → 等待输入
- [ ] `{继续}` 正确触发 NPC/环境动作单元
- [ ] PC 输入正确解析并显示在剧情流中
- [ ] 强制环境插入在连续角色 >= N 时触发
- [ ] 一致性校验在每 K 个动作单元后触发
- [ ] 剧情流清晰区分对话、行动、内心活动
- [ ] 在场人员列表可查看
- [ ] JSON 解析重试逻辑工作正常
